The Problem

47 photos named IMG_2455.HEIC.
You don't know what you own.

Your bookshelf exists. Your library doesn't.
No record of what you've read. No way to filter by country or author gender.
No way to remember if you lent something out.

The books are there. The data isn't.

How it works

One photo. One command.
One entry in your library.

Step 1
Photograph
Snap the cover — iPhone HEIC, JPG, PNG. Point at a folder for batch scanning. Works offline.
Step 2
Claude reads it
Vision extracts title, author, year, country, category, and a one-sentence bio. Uncertain fields stay null — never guessed.
Step 3
Library
Saved to books.json. Synced to Markdown. Ready for the interactive stats dashboard.
Commands

Everything your library needs. Nothing it doesn't.

/books scan <image>
Scan a single cover photo
/books isbn <number>
Look up by ISBN via Open Library — higher accuracy
/books scan-folder <path>
Batch-scan an entire folder of photos
/books list [status]
Terminal overview — no browser needed
/books search <keyword>
Search title, author, country, category
/books rate <title> <1–5>
Star rating — stored and shown in dashboard
/books note <title> <text>
Personal notes — append or replace
/books stats [theme]
Generate interactive HTML dashboard
/books suggest
Next read, based on your library's patterns
/books deploy
Publish to Vercel — permanent sharable URL

Accuracy
80–90%

Cover recognition accuracy.
ISBN lookup pushes it further.

Uncertain fields are left null — never guessed.
Blurry spine? Title stays blank. Lesser-known author? Year is inferred, not confirmed.
Use /books isbn for higher certainty — Open Library returns confirmed metadata, Claude fills the rest.

Sharing

Publish once.
Access anywhere.

The dashboard lives locally — but one command pushes it to a permanent Vercel URL. Same URL every time. Bookmark it on your phone. Run again after adding new books to update.

/books deploy
→ https://my-books-library.vercel.app
